# Project 2 (Due: 05/15/26)

## Overview

For P2, you and your team will create a full stack web application: separate frontend and backend and a complete CI/CD pipeline

## Requirements

- Application Must build and run
- Unit Testing (70% branch coverage for Services and Utilities/Business Logic + 3-5 React tests for front end)
- CI/CD Pipeline on Github Actions
- Implement Secure Authentication (user login / registration with password hashing.)
- Frontend
  - React
  - Styling: plain CSS, Tailwind, bootstrap, etc.
  - Hosted on Azure - MOVED TO STRETCH
- Backend
  - ASP.NET API
  - Docker containerized
  - Hosted on Azure - MOVED TO STRETCH
- SQLServer DB hosted on Azure

## Proposal (Due EOD: 05/05/26)

You'll create a project proposal as a team that includes overview of the project, mvp features, and stretch goals.
**Try to come up with an idea that is not a continuation of P1. If you choose to re-use a group member's idea, your MVP will be larger than if you greenfield a project.**

Include the following in your proposal:

- Project Overview
- MVP features
- ERD
- External API you plan on using (can be called from front-end, back-end or both.)
- Stretch Goals (3-5)
- Project management/work tracking system. (Github Projects suggested, can also use Trello, etc. Stick to free.)

These are not included in the proposal, but do discuss the following:

- Git Repo Practice/Process (merge, code review practice, branch protection, Mono-repo versus 2 repos, etc)
- Coding Convention/Practice
- Communication standard
  - Which medium are we going to use to communicate?
  - At what frequency?

## Technology

- C# (Backend programming language)
- EF Core (ORM)
- SQL Server (Azure hosted)
- ASP.NET Core (Web API Framework)
- xUnit/Moq (Backend Testing)
- Azure (for application hosting)
- React as front end
- Github Actions Pipeline for CICD Pipeline
- Docker for containerization
